Admin Guide

How to manage Tennis String Lab

This page is your quick guide for maintaining the site, updating links, managing players, and handling images.

Shop Listings

How the string shop link works

String result cards now open proshops.html through the Closest Pro Shops button, where nearby shops auto-filter first when the visitor allows browser location or chooses a city from the backup menu.

To add or update a shop, edit proshops.js and update the shop name, city, state or province, region, website, notes, and coordinates.

Pro Shop Review

How to approve submitted listings

New shop requests now come through the form on proshops.html as Pending Admin Review. Nothing goes live automatically.

Review checklist:

1. Open the form submission in Netlify and confirm the request is not spam.
2. In Netlify, turn on form email notifications for the pro-shop-listing-request form so new submissions go to your inbox.
3. Check the shop website, city, and contact details for accuracy.
4. Clean up the specialties and listing notes so the live card reads clearly.
5. If approved, add the shop entry into proshops.js with the correct region, coordinates, and sponsored: true when it is a paid placement.
6. Keep the listing marked as sponsored on the public page when it is a paid placement.
7. Redeploy the site after updating the shop list so the approved listing appears live.

Netlify email note: the site code can submit the form, but the actual new-submission email is controlled inside Netlify under Forms and Notifications.

Pro Players

How to manage pro-player mappings

Use Manage Pro Players to add or delete custom ATP and WTA records. You can save player name, string, tension, racket, and notes there.

Images

How to manage string images

Use String Image Library to drop or paste images onto strings. These are stored in your browser on your computer, so they are useful for local admin work but are not yet a true hosted image system.

Important

Current admin limitation

This is still a static website. The admin pages are separated from the public menu, but they are not protected by login. Real admin-only access would require authentication on the live site.

Deployment

How to publish Tennis String Lab

This is a static website, so the easiest deployment options are Netlify, Cloudflare Pages, GitHub Pages, or a normal web host that accepts HTML, CSS, and JavaScript files.

Basic deployment steps:

1. Keep all project files together in the same folder.
2. Upload the full TennisStringPlanner folder contents to your hosting service.
3. Make sure index.html is the homepage file.
4. Keep app.js, styles.css, proshops.html, proplayers.html, images.html, and admin.html in place.
5. After uploading, test the public homepage and then test the admin pages separately.

Recommended deployment flow:

1. Put the site files into a GitHub repository.
2. Connect that repository to Netlify or Cloudflare Pages.
3. Publish the site and point your domain tennisstringlab.com to the host.
4. Re-upload or redeploy each time you update the site files.

Analytics note: the visit stats page now expects Supabase-backed analytics. Run supabase-analytics.sql in Supabase first, then add SUPABASE_URL and SUPABASE_SERVICE_ROLE_KEY to your Netlify environment variables before testing visit-stats.html.

Important note: browser-saved images and custom local admin records do not automatically move to the live site. Those are stored locally in your browser unless you later build a real backend or file-storage system.

Domain Setup

How to connect tennisstringlab.com

If you use Netlify or Cloudflare Pages, add your site there first and confirm the site is live on the temporary hosting URL.

Then:

1. Add tennisstringlab.com as a custom domain in your hosting dashboard.
2. Follow the host instructions for DNS settings.
3. Update the domain DNS where your domain name was purchased.
4. Wait for DNS to propagate, then test both the root domain and the www version if you want both to work.
